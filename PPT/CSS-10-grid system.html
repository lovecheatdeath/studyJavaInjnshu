今天小课堂分享的内容主要是讲，Grid system（即网格系统）中row container 和 col等是什么个关系？并且简单介绍一下bootstrap。

1.背景介绍

Bootstrap，来自 Twitter，是目前很受欢迎的前端框架。 它由Twitter的设计师Mark Otto和Jacob Thornton合作开发， 是一个CSS/HTML框架。它即是由动态CSS语言Less写成。Bootstrap一经推出后颇受欢迎，一直是GitHub上的热门开源项目。

2.知识剖析

Grid system的核心思想是将屏幕划分成相等大小的格子，一般划分成12,24,或32份。Bootstrap选择默认划分成12份。

规则：1.数据行(.row)必须包含在定宽容器（.container）或满宽容器（.container-fluid）中，以便为其赋予合适的对齐方式和内距(padding)。 2.具体内容应当放置在列容器（column）之内，而且只有列（column）才可以作为行容器(.row)的直接子元素

3.列与列之间有30px的间距。 4.提供4种不同类型的列：

当屏宽小于768px（即小屏）时，列的划分以.col-xs-后面的数字为准。如果没定义.col-xs-，就会变成单列且宽度和row相等。

当屏宽在768px和992px之间（即平板屏）时，列的划分以.col-sm-后面的数字为准。如果没有定义col-sm-，以.col-xs-为准。

当屏宽在992px和1200px之间（即中屏）时，列的划分以.col-md-后面的数字为准。如果没有定义.col-md-，以col-sm-或col-xs-为准。

当屏宽大于1200px（即大屏）时，列的划分以.col-lg-后面的数字为准。如果没有定义.col-md-，以.col-md-或col-sm-或col-xs-为准。
