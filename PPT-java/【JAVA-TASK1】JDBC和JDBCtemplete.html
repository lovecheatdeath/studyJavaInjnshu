1.背景介绍

java的世界就是和数据打交道

框架,和思想的诞生,都基于我们对数据操作的需求

数据库是存储数据的常用工具

JDBC是java应用程序访问数据库的解决方案

JDBC通过定义一套标准API实现访问数据库,对开发者屏蔽了一些细节问题

同时JDBC也被约定为j2ee13大规范之一

技术的发展,JDBC开始显得复杂繁琐

为了简化操作,就诞生了jdbcTemplete

2.知识剖析

JDBC

JDBC(java Database Connectivity): JDBC API为访问不同的数据库提供了一种统一的途径, 就像ODBC一样,JDBC对开发者屏蔽了一些细节问题, 同时,JDBC对数据库的访问也具有平台无关性。

JDBC步骤

加载JDBC驱动程序 → 建立数据库连接Connection → 创建执行SQL的语句Statement → 处理执行结果ResultSet → 释放资源

常用JDBC的API

1.Driver接口:由数据库厂家提供,开发时只需要装载厂商的数据库驱动程序

2.Connection接口:连接特定的数据库,通过指定url,user,password

3.Statement接口:用于执行静态SQL语句并返回生成的结果对象

4.ResultSet接口:用来检索不同类型字段的方法,可以用来遍历结果集

jdbcTemplete

jdbcTemplete是SpringJDBC框架的核心,是Spring在JDBCAPI上定义了一个抽象层，建立的一个JDBC存取的框架

其中涉及的Spring思想有面向Bean编程(BOP)和依赖注入(DI)

面向bean编程核心思想就是IoC是一种设计思想,IoC意味着你设计好的对象交给Spring容器控制,即不创建对象,通过描述创建它们的方式来创建

依赖注入就是容器动态将某个依赖关系注入到组件之中,这样可以提升组件重用的频率
